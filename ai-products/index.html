💡 guancli 是什么？

guancli 是一个通过直接调用 BI 系统 API 来操作观远 BI 的命令行工具。只需 guancli auth login 配置一次认证，之后所有操作——查 ETL、看数据集、搜页面、问 ChatBI、管指标、操作表单——都在终端里一行命令搞定。

🔐 认证方式

  • 用户名密码登录（Token 自动刷新）
  • 直接粘贴 Auth Token
  • 多环境管理（staging/prod 自由切换）
  • ChatBI 复用同一 profile 认证，无需二次配置

📤 输出格式

  • auto 智能表格 / json 结构化数据
  • csv / excel 导出文件
  • expanded 竖排详情 / table 对齐表格
  • --raw 原始 JSON / --brief 轻量输出

📋 十大命令模块

命令做什么典型用法
guancli auth 认证管理：登录、多环境切换、查看状态 guancli auth login
guancli auth list
guancli etl 智能 ETL：目录树、搜索、查看详情（含节点 SQL 与血缘） guancli etl tree
guancli etl get <id>
guancli ds 数据集操作：目录树、搜索、详情、SQL 查询、带筛选预览 guancli ds preview <id> --filter "city EQ 杭州"
guancli page 仪表板/页面：目录树、搜索、详情（含卡片列表与血缘） guancli page get <id>
guancli card 卡片操作：查看元信息、预览数据、导出 CSV/Excel guancli card preview <id> -f csv -o data.csv
guancli chatbi ChatBI 调用：L1 问数 + L2 洞察，支持指定 Agent 模式 guancli chatbi query --theme-name "经营" --message "本月GMV?"
guancli metric 指标管理：搜索、查询、创建/编辑/删除、目录树、公共维度 guancli metric search 营收
guancli metric query <id> --dim 城市
guancli form 表单填报：Airtable 风格 CRUD，支持子表操作 guancli form add <id> --data '{...}'
guancli task 任务排查：查看运行中任务、历史记录、任务详情 guancli task running
guancli app SuperApp：创建项目模板、发布到 BI 平台 guancli app create
guancli app publish

💬 ChatBI —— 自然语言交互式 BI
让业务人员用中文聊天就能完成数据分析，无需学习报表操作
📊

L1 智能问数

输入自然语言问题，ChatBI 自动理解意图、匹配数据主题、生成查询并返回结构化结果。支持时间范围、聚合计算、分组对比等常见分析需求。

guancli chatbi query \ --theme-name "经营主题" \ --message "最近30天各地区营收Top 10"
🔍

L2 智能洞察

不仅回答问题，还会自动分析原因——为什么营收下降？哪个品类拖了后腿？环比变化的主要驱动因素是什么？支持 multi_agent 高级分析模式。

guancli chatbi insight \ --theme-id "theme_123" \ --message "分析营收下降原因" \ --agent-mode single_agent

🎯 指标中心 (Metric Center)
从"各说各话"到"同一种语言"——企业级指标定义、管理与归因平台
1

指标定义标准化

支持 原子指标（最细粒度度量）、复合指标（多指标运算）和 衍生指标（带维度/时间下钻），建立企业统一的指标字典。

2

泛化查询（Advanced Query）

一个指标 ID 即可按任意维度查询——同比、环比、累计、占比、TopN、最近 N 周期……一行命令搞定复杂对比分析。需要 BI 版本 ≥ 8.2.1。

3

指标树归因

建立指标的层级关系树（如 总营收 → 毛利 → 品类毛利 → 单品毛利），自动计算各层级贡献度，一眼看出变化根因。

guancli metric query metric_001 --dim 城市 --compare last_year guancli metric_attribution query tree_001 --period 2026-05
4

公共维度

将多个数据集的相同字段关联为统一的公共维度（如"城市""品类"），一个维度跨数据集使用，保证一致性和可复用性。

5

数据集对指标的反查

给定一个数据集 ID，通过 guancli metric by-dataset <dsId> 一键找出所有使用该数据集的原子指标和下游复合/衍生指标，方便评估数据集变更的影响范围。
